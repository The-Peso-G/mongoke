All the graphql best pratices built in
Powerful queries for all your data
The generated queries follow the mongodb query model, you can do gt, lt, eq, ...
Relay style pagination
Pagination is implemented following the relay style, cursors are available for every field
Authorization with Jwt
You decide what field a user has access to, based on the jwt payload and the document requested
Apollo Federation compliant
You can compose other services together without any effort
Easy deploy via Docker
Lightweight image using docker, generates the service on every deploy in couple of seconds
Features
Model your db types
The configuration describe the db types and connects these to associated collections
The types should represent the objects in the database as close as possible

Authorization with your jwt
You can add python expressions that evaluates to true if you want an user access a resources. The expression is evaluated with the jwt payload data and document data.
The expression are written in Python and evaluated from top down until one evaluates to true
1
2types:
3    User:
4        collection: users
5        guards:
6            -   expression: jwt and jwt['user_id'] == x['_id']
7    BlogPost:
8        collection: posts
9        guards: # not everyone has access to content
10            -   expression: |
11                    jwt and jwt['user_id'] == x['author_id']
12            -   expression: True
13                excluded:
14                    - content
15
Relations between types
Relations are described in the configuration via a query object that will be sent to mongodb, this can evaluate expressions based on the parent component.
1
2relations:
3    - field: posts
4      from: User
5      to: BlogPost
6      relation_type: to_many
7      where:
8          author_id: ${{ parent['_id'] }}
9
Compose your database graphql with other services
Thanks to Apollo federation the mongoke generated service is composable with other graphql services, you can handle mutations and more specific logic in differnt services.
